# Glossar — DE ↔ EN

> Begriffe, die im Curriculum vorkommen, mit deutscher Erklärung und englischem Original-Term. Alphabetisch sortiert.

| Deutsch | Englisch | Erklärung |
|---|---|---|
| Adapter | adapter | Kleine zusätzliche Gewichts-Schicht zum Finetuning (z. B. LoRA). Statt Full Finetuning werden nur die Adapter-Gewichte trainiert. |
| Auftragsverarbeiter | data processor | Eine Stelle, die im Auftrag personenbezogene Daten verarbeitet (Art. 4 Nr. 8 DSGVO). |
| Auftragsverarbeitungsvertrag (AVV) | data processing agreement (DPA) | Vertrag nach Art. 28 DSGVO. |
| Bereitsteller | deployer | Wer ein KI-System unter eigener Verantwortung nutzt (Art. 3 Nr. 4 AI Act). |
| Big Pair Encoding (BPE) | byte-pair encoding | Subwort-Tokenisierungsverfahren (Sennrich et al. 2016). |
| Datenschutz-Folgenabschätzung (DSFA) | data protection impact assessment (DPIA) | Pflicht-Analyse bei hohem Risiko (Art. 35 DSGVO). |
| Diffusion-Modell | diffusion model | Generatives Modell, das aus Rauschen schrittweise zu Bildern (oder anderen Modalitäten) gelangt. |
| Einbettung | embedding | Vektorrepräsentation eines Texts/Bilds. |
| Feinabstimmung | finetuning | Anpassung eines vortrainierten Modells an spezifische Daten. |
| Halluzination | hallucination | Unbelegte/falsche Modell-Aussagen. |
| Hochrisiko-System | high-risk system | KI-System nach AI Act Anhang III oder I. |
| Inferenz | inference | Anwendung eines trainierten Modells zur Vorhersage. |
| KI-Kompetenz | AI literacy | Pflicht nach Art. 4 AI Act. |
| Konformitätsbewertung | conformity assessment | Pflicht für Hochrisiko (Art. 43 AI Act). |
| LLM (großes Sprachmodell) | LLM (large language model) | Modell mit > ~ 1 Mrd. Parametern für Sprachverarbeitung. |
| Mensch-in-der-Schleife | human-in-the-loop | Pattern für AI-Act Art. 14 Human Oversight. |
| Modell-Karte | model card | Strukturierte Doku eines Modells (Mitchell et al. 2018). |
| Prompt-Injection | prompt injection | Angriff, der Prompts manipuliert (OWASP LLM #1). |
| Quantisierung | quantization | Reduktion der Bit-Tiefe eines Modells (z. B. fp16 → int4). |
| Retrieval-Augmented Generation (RAG) | retrieval-augmented generation | Pattern: Retrieval + LLM-Generation. |
| Subwort-Token | subword token | Teil eines Worts in der Tokenisierung. |
| Tokenisierung | tokenization | Aufteilung Text → Tokens. |
| Transparenz-Pflicht | transparency obligation | AI Act Art. 50. |
| Verbote | prohibitions | AI Act Art. 5. |
| Vektor-DB | vector DB | Datenbank für Embeddings (z. B. Qdrant, pgvector). |

## Kurzformen

- **AI Act** = Verordnung (EU) 2024/1689
- **DSGVO** = Datenschutz-Grundverordnung
- **GPAI** = General-Purpose AI
- **NIS2** = Netz- und Informationssystem-Sicherheits­richtlinie 2
- **TDM** = Text and Data Mining
- **TDDDG** = Telekommunikation-Digitale-Dienste-Datenschutz-Gesetz (vormals TTDSG)
